# 第七章 · 深度伪造与鉴别

> **本章定位**：深度伪造（Deepfake）已经从"实验室玩具"变成"社会风险"。  
> 本章从技术原理、社会影响、典型案例三个层面展开，重点讲解**语音克隆**与**图像伪造鉴别**。

---

## 7.1 深度伪造（Deepfake）概述

### 📖 概念铺垫

> 💡 **"眼见为实"的失效**：  
> 2017 年，Reddit 用户 *deepfakes* 首次将名人面孔合成到色情视频中。  
> 2019 年，AI 语音克隆让一家英国能源公司 CEO"开口"批准转账 24 万美元。  
> 2023 年，伪造的"教皇穿羽绒服"图片在 Twitter 上数千万次曝光。  
> 2024 年，多国选举季被深度伪造视频搅动。

> **深度伪造**（Deepfake，Deep + Fake）是指用深度学习技术生成的伪造内容，  
> 主要包括：图像伪造、视频伪造、语音克隆、文本伪造。

### 🔍 原理深挖

#### 7.1.1 三大主流技术路线

| 技术 | 代表 | 优势 | 局限 |
|---|---|---|---|
| **自编码器** | Deepfake 原版、Real-Time-Voice-Cloning | 简洁、可解释 | 质量受限 |
| **GAN** | StyleGAN、FaceSwap-GAN | 图像质量高 | 训练不稳定 |
| **扩散模型** | Stable Diffusion、Midjourney、DALL·E | 质量高、可控 | 计算量大、潜在滥用风险大 |
| **Transformer** | AudioLM、UniAudio | 长序列建模 | 数据需求大 |

#### 7.1.2 深度伪造的攻击面

| 模态 | 典型场景 | 危害 |
|---|---|---|
| 图像 | 政客假演讲、明星虚假新闻 | 舆论操控 |
| 视频 | 伪造监控、伪造"犯罪现场" | 司法冤案、勒索 |
| 语音 | 实时语音克隆、电话诈骗 | 财产损失、声纹滥用 |
| 文本 | AI 生成假新闻、垃圾邮件 | 信任危机 |
| 多模态 | "假视频 + 假声音 + 假字幕" | 综合性欺骗 |

## 7.2 语音克隆

### 📖 概念铺垫

> 基于开源项目 [Real-Time-Voice-Cloning](https://github.com/CorentinJ/Real-Time-Voice-Cloning)。

> **场景**：3 秒样本即可克隆一个声纹，合成"以假乱真"的语音。

> 💡 **类比**：  
> 以前的语音合成像"念稿"，需要专业录音棚 + 大量样本。  
> 现在的 Real-Time-Voice-Cloning 像"模仿秀"——只要听对方说几句话，就能"学他说话"。

### 🔍 原理深挖

#### 7.2.1 三段式架构

Real-Time-Voice-Cloning 的核心是**Encoder + Synthesizer + Vocoder** 三段式架构：

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 参考音频 │ ─▶ │  Encoder │ ─▶ │ Speaker  │ ─┐
│ "5秒样" │    │ (说话人  │    │ Embedding│  │
└──────────┘    │  编码器) │    └──────────┘  │
                └──────────┘                  │
                                               ▼
                              ┌────────────────────────────┐
                              │       Synthesizer          │
                              │  (Tacotron2: 文本 → Mel)    │
                              └────────────────────────────┘
                                               │
                                               ▼
                                       ┌──────────┐
                                       │  Mel     │ ─▶ 语音波形
                                       │  Spectro │    (Vocoder)
                                       └──────────┘
```

1. **Encoder**：把任意长度的参考音频映射到一个**说话人嵌入向量**（speaker embedding）。
2. **Synthesizer**：把"要说的文本 + 说话人嵌入"合成出**Mel 频谱**。
3. **Vocoder**：把 Mel 频谱转成可听的**语音波形**。

> **核心洞察**：说话人嵌入是"声纹的指纹"——5 秒就足够抽取一个稳定指纹。

#### 7.2.2 各模块的"实现机制"

| 模块 | 模型 | 训练数据 | 输出 |
|---|---|---|---|
| **Encoder** | LSTM-based GE2E | VoxCeleb1/2（数千说话人） | 256 维嵌入 |
| **Synthesizer** | Tacotron 2 | LJSpeech + 自建 | 80 维 Mel 频谱 |
| **Vocoder** | WaveRNN | LJSpeech | 16kHz 波形 |

### 🛠️ 实战操作

#### 7.2.3 关键代码片段：完整克隆流程

```python

class VoiceSynthesis(object):
    def __init__(self, encoder_weights, vocoder_weights, syn_dir):
        # 1) 加载 Encoder（说话人嵌入提取）
        from encoder import inference as encoder
        encoder.load_model(encoder_weights)
        self.encoder = encoder
        # 2) 加载 Synthesizer（文本 → Mel）
        from synthesizer.inference import Synthesizer
        self.synthesizer = Synthesizer(syn_dir)
        # 3) 加载 Vocoder（Mel → 波形）
        from vocoder import inference as vocoder
        vocoder.load_model(vocoder_weights)
        self.vocoder = vocoder

    def synthesize(self, ref_audio_path, text, dst_path):
        # 1) 提取参考音频的说话人嵌入
        wav = self.encoder.preprocess_wav(ref_audio_path)
        embed = self.encoder.embed_utterance(wav)
        # 2) 用 Synthesizer 合成 Mel
        specs = self.synthesizer.synthesize_spectrograms([text], [embed])
        spec = specs[0]
        # 3) 用 Vocoder 把 Mel 转成波形
        from vocoder import inference as vocoder
        gen_wav = vocoder.infer_waveform(spec)
        # 4) 写到文件
        import scipy.io.wavfile as wavfile
        wavfile.write(dst_path, 16000, gen_wav)
```

> **要点**：
> - 三段式解耦后，**Encoder 可独立使用**（如做声纹识别）。
> - **Synthesizer + Vocoder 联合**完成 TTS。
> - 端到端推理可在 **CPU 上实时运行**（5 秒样本输入 → 流式输出语音）。

#### 7.2.4 关键代码片段：Encoder 提取说话人嵌入

```python

def embed_utterance(wav, using_partials=True, return_partials=False):
    """
    输入：音频波形
    输出：说话人嵌入向量（256 维）
    """
    # 1) 滑窗切分（160ms 窗口）
    frames = wav_to_mel_spectrogram(wav)  # shape: (N, 80, 40)
    # 2) 送入 SpeakerEncoder
    embed = _model.forward(frames)         # shape: (256,)
    # 3) 长度归一化
    embed = embed / torch.norm(embed)
    return embed
```

> **关键点**：LSTM-based GE2E 模型**对每个窗口都输出嵌入**，
> 再用"中心聚类"取最终嵌入——5 秒音频通常得到 50+ 个窗口。

### 💡 思考与拓展

> **真实事件**：
> - **2019 英国 CEO 语音诈骗**：损失 24 万美元。攻击者用 AI 合成 CEO 德语口音英语。
> - **2023 深度伪造视频在 TikTok 爆发**：多个明星"代言"从未发布的产品。
> - **2024 美国大选季深度伪造**：AI 拜登电话劝退选民事件震动全美。

> **思考题**：
> 1. 实时语音克隆如何被滥用于**客服诈骗**？  
> 2. **声纹活体检测**能防御吗？  
> 3. 法规上，能否要求"AI 生成语音必须添加水印/显式标识"？

> **防御技术**：
> - **活体检测**：让对方做一个**随机面部表情/口型同步**任务。
> - **声纹水印**：原始录制时嵌入水印，AI 合成无法复制。
> - **语义一致性检测**：检测语音内容是否"上下文合理"。
> - **法规应对**：欧盟 AI Act、Deepfake 透明度法等。

## 7.3 深度伪造鉴别

### 📖 概念铺垫


如果说 7.2 是"造假的矛"，那 7.3 就是"鉴别的盾"。  
Antifakes 是一个**基于 EfficientNet 的图像伪造鉴别器**。

> 💡 **类比**：  
> 鉴别器就像"验钞机"——造假者用高级材料、防伪线、UV 图案层层包装，  
> 鉴别器要找出"哪一处和真钞对不上"。

### 🔍 原理深挖

#### 7.3.1 图像伪造的"指纹"

不同生成方法在图像中留下的"指纹"不同：

| 生成方法 | 指纹特征 |
|---|---|
| **StyleGAN** | 频域伪影、肤色异常 |
| **扩散模型** | 纹理重复、细节平滑 |
| **FaceSwap** | 边界伪影、眨眼频率异常 |
| **Deepfake** | 嘴型与音频不同步、瞳孔反射异常 |

#### 7.3.2 鉴别器的技术路径

| 路径 | 解释 | 优势 | 局限 |
|---|---|---|---|
| **像素统计** | RGB / HSV 像素分布异常 | 简单 | 容易过拟合 |
| **频域特征** | FFT / DCT 高频能量异常 | 对 GAN 有效 | 扩散模型难 |
| **生物信号** | 眨眼频率、瞳孔、脉搏 | 物理可解释 | 容易绕过 |
| **深度学习** | CNN / ViT 端到端 | 准确率高 | 泛化性差（"今天的鉴别器识别不出明天的假图"） |
| **多模态** | 音视频一致性 | 鲁棒 | 计算量大 |

### 🛠️ 实战操作

#### 7.3.3 关键代码片段：Antifakes 鉴别器

```python

import torch
from torchvision import transforms
from .model import EfficientNet

class AntiFakes(object):
    def __init__(self, model_path, arch='efficientnet-b4', classes_num=2):
        # 1) 加载预训练 EfficientNet
        self.model = EfficientNet.from_pretrained(arch, num_classes=classes_num)
        self.model.load_state_dict(torch.load(model_path, map_location='cpu'))
        self.model.eval()
        # 2) 预处理
        self.transform = transforms.Compose([
            transforms.Resize((224, 224)),
            transforms.ToTensor(),
            transforms.Normalize([0.485, 0.456, 0.406], [0.229, 0.224, 0.225]),
        ])

    def predict(self, image_path):
        """
        返回 (label, confidence)：
          - 0 = 真
          - 1 = 假
        """
        from PIL import Image
        img = Image.open(image_path).convert('RGB')
        x = self.transform(img).unsqueeze(0)
        with torch.no_grad():
            logits = self.model(x)
        probs = torch.softmax(logits, dim=1)
        label = probs.argmax().item()
        return label, probs[0][label].item()

    def predict_video(self, video_path, frame_distance=5):
        """
        抽帧并对每帧预测，做投票
        """
        import cv2
        cap = cv2.VideoCapture(video_path)
        preds = []
        frame_idx = 0
        while True:
            ret, frame = cap.read()
            if not ret: break
            if frame_idx % frame_distance == 0:
                img = Image.fromarray(cv2.cvtColor(frame, cv2.COLOR_BGR2RGB))
                label, _ = self.predict(img)  # 注意：实际实现要先保存再读取
                preds.append(label)
            frame_idx += 1
        # 投票
        if not preds: return None
        from collections import Counter
        return Counter(preds).most_common(1)[0][0]
```

> **关键点**：
> - 使用预训练 **EfficientNet** 提取高层特征。
> - 视频鉴别时**抽帧 + 投票**，降低单帧误判。
> - 帧距 `frame_distance=5` 是经验值（平衡速度与准确率）。

#### 7.3.4 鉴别效果

| 训练数据 | 训练样本数 | 在 FaceForensics++ 上准确率 |
|---|---|---|
| FaceForensics++（Deepfakes） | ~5000 视频 | 95%+ |
| FaceForensics++（FaceSwap） | ~5000 视频 | 92% |
| Real vs Diffusion 生成 | 混合 | 70–80% |
| 跨数据集测试 | 训练外 | 60–70% |

> **跨数据集准确率下降**是当前最大挑战——今天的鉴别器训练时没见过明天的假图。

### 💡 思考与拓展

> **思考题**：
> 1. 鉴别器和生成器是"猫鼠游戏"——生成器不断进化，鉴别器怎么办？  
> 2. 能否用**自监督预训练 + 微调**让鉴别器更鲁棒？  
> 3. **多模态（音+视）联合鉴别**是未来吗？  
>    （提示：Deepfake 经常出现"嘴型对不上声音"，这是关键指纹）

> **行业工具**：
> - **FaceForensics++**（2019）：大规模 Deepfake 数据集。
> - **DFDC**（Facebook Deepfake Detection Challenge）：百万级数据。
> - **FakeCatcher**（Intel）：实时伪造检测，宣称为 96% 准确率。

## 7.4 章节小结

| 主题 | 核心攻击 | 核心防御 |
|---|---|---|
| 深度伪造概述 | 自编码器、GAN、扩散模型 | 内容水印、来源凭证 |
| 语音克隆 | Encoder + Synthesizer + Vocoder | 声纹活体、语义一致性 |
| 图像鉴别 | EfficientNet、Transformer | 多模态、可解释 AI |
| **风险延伸（→ 第八章）** | **多模态 Agent 看到伪造内容被操纵** | **多模态鉴别 + 跨模态一致性** |

> **核心要点**：
> 1. **生成与鉴别是永无止境的对抗**——今天最强的模型明天可能被超越。
> 2. **技术之外需要制度**——欧盟 AI Act、美国《深度伪造责任法案》、中国《生成式 AI 服务管理办法》。
> 3. **水印 + 来源凭证**是产业化的"轻量级"防御。
> 4. **多模态联合**是未来鉴别方向。
> 5. 🆕 **当 Agent 拥有"看图 / 听音"能力**（如 Operator / Computer-Use），第七章的鉴别能力是 Agent 输入层防御的关键补充——详见 [第八章 8.3.4 多模态注入](08-第八章-AI Agent安全：威胁、攻防与防御.md)。

## ⚠️ 风险提示

- 实施深度伪造**严禁**用于欺诈、勒索、虚假信息、政治操控。
- 真实业务中部署鉴别器时，要**关注误报率**——把正常内容错判为伪造会引发投诉。
- 鉴别工具是**辅助手段**，不应作为唯一决策依据。
- 在大模型时代，**人类对内容真实性的判断力正在被侵蚀**——这要求教育与立法并举。
