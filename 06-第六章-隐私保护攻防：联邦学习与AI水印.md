# 第六章 · 隐私保护攻防：联邦学习与 AI 水印

> **本章定位**：隐私不是"加密数据库"就能解决的——  
> 模型本身也可能"记住"训练数据。  
> 本章先梳理隐私威胁模型，再深入**联邦学习梯度窃取攻击**与**AI 对抗水印**两种代表性攻防手段。

---

## 6.1 隐私威胁与防御总览

### 📖 概念铺垫

> 💡 **为什么"数据脱敏"还不够？**  
> 即使把训练数据中的人名、地址替换为假名，模型也可能**间接记住**个体特征。  
> 攻击者可以通过**模型反演、成员推断、训练数据提取**等方式"反向还原"隐私。

四类典型隐私攻击：

| 攻击 | 解释 | 输出 |
|---|---|---|
| **成员推断** Membership Inference | 推断某条数据是否在训练集中 | 隐私的"开 / 关" |
| **模型反演** Model Inversion | 还原与某条输出对应的输入 | 训练样本"近似" |
| **属性推断** Attribute Inference | 推断训练集的整体统计属性 | 数据集的"画像" |
| **训练数据提取** Training Data Extraction | 还原训练样本的逐字内容 | 真实个人信息 |

> 本章聚焦**梯度窃取**（模型反演的一种）和**AI 水印**（一种主动防御）。

### 🔍 原理深挖

#### 6.1.1 隐私的形式化定义：差分隐私

最严格的隐私定义是**差分隐私（Differential Privacy, DP）**：

$$
\Pr[\mathcal{M}(D) \in S] \le e^{\epsilon} \cdot \Pr[\mathcal{M}(D') \in S]
$$

其中 $D'$ 与 $D$ 相差一条样本，$\epsilon$ 是"隐私预算"——越小越隐私，但模型效用越差。

> 直观：去掉/加上某条样本，模型输出分布**几乎不变**——这意味着**没有任何一条样本的"贡献"被模型单独记忆**。

#### 6.1.2 防御层次

```
┌──────────────────────────────────┐
│ 1. 数据层：脱敏 / 差分隐私合成数据  │
├──────────────────────────────────┤
│ 2. 训练层：DP-SGD、联邦学习、安全聚合│
├──────────────────────────────────┤
│ 3. 模型层：知识蒸馏、模型水印       │
├──────────────────────────────────┤
│ 4. 输出层：内容过滤、输出限速       │
└──────────────────────────────────┘
```

## 6.2 联邦学习基础

### 📖 概念铺垫

> 💡 **"数据不动模型动"**：  
> 传统做法是把用户数据上传到云端训练。  
> 联邦学习（Federated Learning）让数据**留在本地**，只把**模型梯度**上传给中心服务器聚合。

```
                  中心服务器
                 （聚合梯度）
                /     |     \
            客户端1  客户端2  客户端3
            (本地数据) (本地数据) (本地数据)
            (本地训练) (本地训练) (本地训练)
            (上传梯度) (上传梯度) (上传梯度)
```

### 🔍 原理深挖

#### 6.2.1 FedAvg 算法

McMahan 等人 2017 年提出的经典联邦平均算法：

$$
\theta_{t+1} = \theta_t - \eta \cdot \frac{1}{K} \sum_{k=1}^{K} g_k
$$

其中 $g_k$ 是客户端 $k$ 在自己数据上算出的梯度，$K$ 是参与的客户端数。

#### 6.2.2 联邦学习的"安全幻觉"

> **重要事实**：  
> 联邦学习**不是自动隐私保护**。  
> "梯度"看似抽象，但攻击者可以**从梯度反推原始数据**。

这就是 6.3 节要讨论的**梯度窃取攻击**。

## 6.3 梯度数据窃取攻击

### 📖 概念铺垫


Zhu 等人 2019 年发表 *Deep Leakage from Gradients*，震撼了联邦学习领域：
**直接从客户端上传的梯度中还原出原始训练图像**——连 MNIST 数字都能像素级还原。

> 💡 **类比**：  
> 你以为上传"成绩单"（梯度）比上传"试卷"（数据）更安全。  
> 攻击者拿到成绩单后，反推试卷内容——"他数学 95，那么他选择题错了 1 道，填空错了 1 道……"，  
> 几次迭代就还原出原题。

### 🔍 原理深挖

#### 6.3.1 攻击模型

- **攻击者**：中心服务器（或中间人）。
- **已知**：模型参数 $\theta$、客户端上传的梯度 $g$。
- **目标**：还原客户端的输入 $x$ 和标签 $y$。

#### 6.3.2 优化目标

$$
\min_{x', y'} \;\; \mathcal{L}_{sim}\left( \nabla_\theta \mathcal{L}_{CE}(f_\theta(x'), y'), \; g \right)
$$

其中 $\mathcal{L}_{sim}$ 是"梯度相似度"——通常用 L2 距离或余弦相似度。

直观：从**随机噪声**出发，反复调整 $x', y'$，使它们产生的梯度**与真实梯度 $g$ 越来越接近**。

#### 6.3.3 攻击流程

```
┌────────────────┐    ┌────────────────┐    ┌────────────────┐
│ 初始化 x' 随机  │ ─▶ │ 计算 g' = ∂L/∂θ│ ─▶ │ 计算距离 ||g-g'||│
└────────────────┘    └────────────────┘    └────────────────┘
                                                    │
                                                    ▼
                                            ┌────────────────┐
                                            │ 反向传播到 x'  │
                                            │ x' ← x' - lr·∇│
                                            └────────────────┘
                                                    │
                                                    └──────────▶ 循环
```

> 通常 100–200 次迭代就能在 MNIST 上像素级还原原图。

### 🛠️ 实战操作

#### 6.3.4 关键代码片段：梯度窃取攻击

```python

import torch
import torch.nn as nn
import torchvision

class LeNet(nn.Module):
    """被攻击的 LeNet"""
    def __init__(self):
        super().__init__()
        self.body = nn.Sequential(
            nn.Conv2d(3, 12, kernel_size=5, padding=2, stride=2), nn.Sigmoid(),
            nn.Conv2d(12, 12, kernel_size=5, padding=2, stride=2), nn.Sigmoid(),
            nn.Conv2d(12, 12, kernel_size=5, padding=2, stride=1), nn.Sigmoid(),
        )
        self.fc = nn.Linear(768, 10)
    def forward(self, x):
        x = self.body(x)
        x = x.view(x.size(0), -1)
        return self.fc(x)

def attack_leakage(gt_gradients, model, input_shape=(3, 32, 32), num_classes=10,
                   iters=200, lr=0.1):
    """
    梯度窃取攻击：从 gt_gradients 还原原始输入
    """
    # 1) 随机初始化"假输入"
    dummy_x = torch.randn(input_shape, requires_grad=True)
    dummy_y = torch.randn(num_classes, requires_grad=True)
    optimizer = torch.optim.Adam([dummy_x, dummy_y], lr=lr)
    crit = nn.CrossEntropyLoss()

    for i in range(iters):
        # 2) 用"假输入"再算一次梯度
        pred = model(dummy_x)
        loss = crit(pred, dummy_y.argmax())
        grad_x = torch.autograd.grad(loss, model.parameters(), create_graph=True)
        # 3) 计算与真实梯度的距离
        diff = sum(((gx - gtx) ** 2).sum() for gx, gtx in zip(grad_x, gt_gradients))
        # 4) 反向传播更新 dummy_x / dummy_y
        optimizer.zero_grad()
        diff.backward()
        optimizer.step()
    return dummy_x.detach(), dummy_y.detach()
```

> **要点**：
> - `create_graph=True` 让二阶导可用，便于优化 $x'$。
> - Adam + 适当学习率（0.1）通常能稳定收敛。
> - 100–200 步即可在 MNIST 上像素级还原，CIFAR-10 上**接近像素级**。

#### 6.3.5 攻击效果

| 数据集 | 平均 PSNR | 视觉相似度 | 还原率 |
|---|---|---|---|
| MNIST | ~50 dB | 像素级一致 | 100% |
| CIFAR-10 | ~30 dB | 视觉可辨 | 95% |
| ImageNet (32×32) | ~25 dB | 语义可辨 | 80% |

### 💡 思考与拓展

> **思考题**：
> 1. 为什么小 batch（如 batch=1）下攻击成功率最高？  
>    （提示：batch 大时梯度被"平均"，每个样本的"指纹"被稀释）
> 2. 攻击需要**白盒访问**模型吗？  
>    （提示：在联邦学习中，服务器**天然有**模型，攻击可行）

> **防御方法**：
> - **梯度裁剪（Gradient Clipping）**：限制单次梯度的最大范数，但部分研究显示对高级攻击效果有限。
> - **差分隐私噪声**：在梯度上加高斯噪声，破坏还原。
> - **安全聚合（Secure Aggregation）**：用密码学让服务器只看到"聚合后的梯度"，看不到单客户端。
> - **梯度稀疏化**：只上传 top-k 重要梯度。
> - **数据脱敏**：本地训练前对输入做模糊/裁剪。

## 6.4 利用 AI 水印对抗隐私泄露

### 📖 概念铺垫


传统数字水印（如可见 logo、频域 LSB）依赖"人眼能识别"。
**AI 水印**是**为 AI 模型设计的水印**——人眼几乎看不见，但能**让图像识别模型无法正确分类**。

> 💡 **类比**：  
> 你在朋友圈发"娃的照片"前，先用 AI 加一层"对抗扰动"。  
> 结果：人眼看到的娃依然可爱，但爬取你照片训练"亲子识别模型"的科技公司**训练不出有效模型**。

### 🔍 原理深挖

#### 6.4.1 传统水印 vs AI 水印

| 维度 | 传统水印 | AI 水印 |
|---|---|---|
| 目的 | 版权追溯 / 标识来源 | **阻止 AI 训练 / 误分类** |
| 可见性 | 半可见 / 不可见 | 几乎不可见 |
| 触发对象 | 人眼 / 特定算法 | AI 模型 |
| 实现 | 频域变换 | 对抗扰动（FGSM / PGD） |
| 副作用 | 影响观看 | 几乎不影响观看 |

#### 6.4.2 攻击 / 防御双刃剑

> **关键洞察**：AI 水印**本质上是对抗样本**。
>
> - 把它用在"保护自己照片不被 AI 训练"上 = **防御**。
> - 把它用在"让你的照片骗过人脸识别"上 = **攻击**。
>
> **同一段代码，两种用途**——这是 AI 安全里最值得警惕的"双刃剑"之一。

### 🛠️ 实战操作

#### 6.4.3 关键代码片段：Inception v3 上的对抗水印

```python

import tensorflow as tf
import tensorflow.contrib.slim as slim
import tensorflow.contrib.slim.nets as nets

# 1) 加载 Inception v3
sess = tf.InteractiveSession()
image = tf.Variable(tf.zeros((299, 299, 3)))
logits, probs = inception(image, reuse=False)  # 前向传播
# 恢复预训练权重
saver = tf.train.Saver([v for v in tf.global_variables() if v.name.startswith('InceptionV3/')])
saver.restore(sess, 'inception_v3.ckpt')

# 2) 加载 ImageNet 类别名
with open('imagenet.json') as f:
    imagenet_labels = json.load(f)

# 3) 定义"目标类别"：让模型把"猫"误识别为"杯子"
TARGET_CLASS = imagenet_labels.index('cup')

# 4) 对抗扰动优化
learning_rate = 0.1
for i in range(100):
    # 计算损失：让目标类概率最大化
    loss = tf.nn.softmax_cross_entropy_with_logits(
        labels=tf.one_hot(TARGET_CLASS, 1001), logits=logits
    )
    # 对输入求梯度
    grad = tf.gradients(loss, image)[0]
    # 沿梯度方向更新图像
    noise = tf.sign(grad) * learning_rate
    image = tf.clip_by_value(image + noise, 0, 255)
sess.run(tf.global_variables_initializer())
# 5) 输出最终带水印的图像
```

> **关键点**：
> - 目标类是攻击者指定的（如 "cup"）。
> - 扰动幅度由 `learning_rate` 控制。
> - 100 步左右可获得稳定误分类。

#### 6.4.4 防御 / 攻击的两种用途

| 用途 | 攻击者 | 目标 | 效果 |
|---|---|---|---|
| **保护隐私** | 数据所有者 | 阻止 AI 爬取训练 | 爬取的数据"训练不出"有效模型 |
| **实施攻击** | 攻击者 | 让人脸识别失效 | 安防系统失效、误识别 |

### 💡 思考与拓展

> **思考题**：
> 1. AI 水印能否被"清洗"？  
>    （提示：JPEG 压缩、轻度模糊可能破坏扰动，但会降低图片质量）
> 2. AI 水印是否对**所有模型**有效？  
>    （提示：迁移性是关键挑战，常见做法是 ensemble 多个模型优化扰动）
> 3. 法律上，发对抗水印照片"合理"吗？  
>    （提示：欧美判例倾向于"合理使用"，但仍存争议）

> **行业现状**：
> - **Glaze**（2023）：芝加哥大学发布，专为艺术家作品设计，干扰 AI 风格模仿。
> - **PhotoGuard**（2023，MIT）：用扩散模型生成更鲁棒的扰动。
> - **Adobe Content Credentials**：C2PA 联盟推动"内容来源凭证"标准，互补于 AI 水印。

## 6.5 章节小结

| 主题 | 核心攻击 | 核心防御 |
|---|---|---|
| 隐私威胁总览 | 成员推断、模型反演、属性推断、训练数据提取 | 差分隐私、联邦学习、AI 水印 |
| 联邦学习基础 | 梯度窃取 | DP-SGD、安全聚合、梯度裁剪 |
| 梯度窃取 | 优化"假输入"产生与真梯度相似的输出 | 噪声注入、加密聚合、稀疏化 |
| AI 水印 | 对抗扰动让模型失效 | JPEG 压缩（部分有效）；更多需要**多模型集成扰动** |

> **核心要点**：
> 1. **联邦学习 ≠ 自动隐私**。梯度反推是真实的威胁。
> 2. **AI 水印是把双刃剑**——既能保护隐私也能实施攻击。
> 3. 隐私保护需要**数据 + 训练 + 模型 + 输出**四层联动。
> 4. **差分隐私**目前是最严格的隐私保证，但代价是模型效用下降 2–10%。

---

## ⚠️ 风险提示

- 在生产环境中**严禁**对真实用户的训练数据发起梯度窃取攻击。
- 部署联邦学习时，**至少**应同时启用**梯度裁剪 + 差分隐私 + 安全聚合**。
- AI 水印**不应**作为唯一防御手段，应与**法律、技术、运营**综合配合。
