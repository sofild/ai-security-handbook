# 前言：AI 安全的时代背景与本书使命

## 0.1 一个让人不安的早晨

2018 年，一篇题为 *Accessorize to a Crime: Real and Stealthy Attacks on State of the Art Face Recognition* 的论文在 ACM CCS 上发表。
研究者用一副造价不到 1 美元的眼镜框，就成功骗过了当时业内顶尖的人脸识别系统。
2019 年，研究者用一段 5 秒的音频样本克隆出企业 CEO 的声音，指挥其下属向指定账户汇款，制造了一起震惊业界的"AI 语音诈骗"案。
2023 年，一段伪造的"五角大楼爆炸"图片在 Twitter 上一度造成股市短时下跌。
2024 年，深度伪造视频在选举季泛滥，多国监管机构被迫连夜发布"AI 紧急禁令"。

这些事件指向一个事实：**当 AI 从实验室走入社会，它的安全问题也随之从论文走入现实**。

## 0.2 什么是 AI 安全

"AI 安全"（AI Security）是一个含义丰富的概念，本书中我们沿用学界与工业界通用的两层划分：

1. **AI Security（对抗性安全）**：把 AI 系统视为被攻击目标，关注**对抗样本、后门、数据投毒、模型窃取、隐私推断、深度伪造**等技术攻防。
2. **AI Safety（稳健性 / 价值对齐）**：关注 AI 系统的**鲁棒性、可解释性、公平性、价值观对齐**，让 AI 在长尾场景和分布外数据下表现可信。

本书聚焦在 **AI Security**（同时兼顾部分 Safety 主题，如偏见、隐私），并把内容沿着"攻击 → 防御"的脉络展开。

## 0.3 为什么写这本书

我们浏览了大量相关教程后，发现一个共性问题：

- **教程过"散"**：很多内容以代码片段呈现，缺乏原理讲解与威胁模型梳理。
- **教程过"专"**：深度学习研究者关注的"对抗样本"和产业界关注的"内容风控"互不连通。
- **教程过"技术"**：缺少法律、伦理、行业实践的视角。

因此，本书的定位是：

> **以代码实战为锚点，向外延伸必要的前置知识、原理推导、防御方法、行业案例，
> 让零基础到从业者的读者都能找到适合自己的阅读路径。**

## 0.4 本书如何阅读

为照顾不同层次读者，本书采用统一的模块化标签体系：

- 📖 **概念铺垫**：用通俗语言 + 生活化类比解释核心概念。
- 🔍 **原理深挖**：数学公式、算法推导、攻击/防御的内在机制。
- 🛠️ **实战操作**：基于开源项目代码，摘录**关键代码片段**配合讲解。
- 💡 **思考与拓展**：行业真实事件、防御方案、开放问题、推荐阅读。
- ⚠️ **风险提示**：法律合规、伦理边界、研究伦理。

读者可以按"读者层次 × 标签"组合进行阅读（详见 [README](README.md) 中的阅读路径表）。

## 0.5 本书约定

- **代码语言**：代码示例主要使用 Python 3 + PyTorch / TensorFlow。
- **运行环境**：建议使用 Anaconda 或 venv 隔离环境，各小节依赖略有不同。
- **数学公式**：使用 LaTeX 语法，渲染失败时不影响文字阅读。
- **代码片段**：代码片段为相关开源项目源码的精简版。
- **数据 / 模型**：涉及的数据与模型请遵守对应开源项目的 LICENSE。
- 🆕 **覆盖范围**：本书记录 2020–2026 年间主流攻防技术；2026 年版**新增第八章 · AI Agent 安全**，覆盖 2025–2026 年最显著的产业演进（ReAct / MCP / Computer-Use / OWASP Agentic AI）。
- 🆕 **代码呈现**：第八章以伪代码片段 + 行业事件 + 主流方案为主线，不依赖外部代码目录。读者可在 LangGraph / AutoGen / CrewAI 中按伪代码思路复现。

## 0.7 一个开放邀请

AI 安全是一个**仍在快速演化**的领域，本书记录的是 2020–2025 年间主流攻防技术的"切片"。
当您读到某一章时，请思考：

- 这个攻击的**根本成因**是什么？是数据？模型架构？训练目标？还是部署流程？
- 如果我是**系统设计者**，我能在哪个环节加入防御？
- 如果我是**监管者**，我应如何平衡创新与风险？

> 攻防无止境。**问题比答案更重要。**
