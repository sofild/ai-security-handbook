## 2.1 对抗样本：从"熊猫+噪声=长臂猿"讲起

### 📖 概念铺垫

2014 年，Ian Goodfellow 等人在论文 *Explaining and Harnessing Adversarial Examples* 中展示了著名的"熊猫"案例：

> 一张被高置信度识别为"熊猫"的图像，叠加了一小段**人眼几乎不可见**的扰动后，被误分类为"长臂猿"，且置信度高达 99.3%。

这启发了后续十年的研究：**深度神经网络对精心设计的输入异常脆弱**。

> 💡 **生活化类比**：  
> 对抗样本就像考试中的"出其不意"。模型平时训练时只会见到"教科书题型"，但攻击者会出"脑筋急转弯"。  
> 关键差别在于：人脑可以泛化到题型变化，模型的"决策边界"却很脆弱。

### 🔍 原理深挖

#### 2.1.1 数学定义

设模型为 $f_\theta: \mathcal{X} \to \mathcal{Y}$，原始输入为 $x$，真实标签为 $y$，$f_\theta(x) = y$。  
**对抗样本** $x'$ 满足：

$$
x' = x + \delta,\quad \|\delta\|_p \le \epsilon,\quad f_\theta(x') \ne y
$$

其中：

- $\delta$ 是加在原始输入上的**扰动**（perturbation）。
- $\|\delta\|_p$ 是某种**范数约束**，限制扰动大小以保证"人眼几乎看不见"。
- $\epsilon$ 是攻击者能容忍的最大扰动幅度。

#### 2.1.2 常见范数的几何意义

| 范数 | 公式 | 几何含义 | 实际意义 |
|---|---|---|---|
| L0 | $\lVert\delta\rVert_0 = \vert\{i: \delta_i \ne 0\}\vert$ | 改变像素的**数量** | 几张贴纸就能骗人 |
| L2 | $\lVert\delta\rVert_2 = \sqrt{\sum \delta_i^2}$ | 欧氏距离（全局能量） | 整体不要太亮 |
| L∞ | $\lVert\delta\rVert_\infty = \max_i \vert\delta_i\vert$ | 任何单像素的最大变化 | 噪声均匀分布 |

> L∞ 在视觉上最易控制（每像素改一点点），是研究最广泛的范数；L0 接近物理补丁攻击，第七章会再次相遇。

#### 2.1.3 攻击与防御的"猫鼠循环"

```
1990s–2010：垃圾邮件检测绕过（早期对抗性研究）
2013：   Szegedy et al. 发现深度神经网络的反直觉脆弱性
2014：   Goodfellow et al. 提出 FGSM，给出线性解释
2017：   Madry et al. 提出 PGD / 对抗训练（公认最强防御）
2018：   Carlini & Wagner (C&W) 攻击打破许多"看似鲁棒"的防御
2019+：  物理世界、跨模态、可认证鲁棒性 兴起
```

> 攻防博弈像一个弹簧：**新攻击 → 新防御 → 新攻击 → 新防御**。  
> 这也是本书反复出现的主题。

### 💡 思考与拓展

> 思考题：  
> 1. 如果你是一个安全工程师，你会选择"提高准确率"还是"提高鲁棒性"？  
> 2. **鲁棒性**是否一定意味着**准确率下降**？在哪些场景下可以兼得？

## 2.2 图像识别引擎绕过：FGSM

### 📖 概念铺垫

> **场景**：在 MNIST 手写数字识别上，让 LeNet 把"7"识别为"3"。

FGSM（Fast Gradient Sign Method）由 Goodfellow 等人在 2014 年提出，是**最简单也最经典的对抗样本生成方法**。  
它的核心思想一句话：**沿着损失函数对输入的梯度方向，施加微小的扰动**。

### 🔍 原理深挖

#### 2.2.1 公式推导

我们希望找到 $x'$，使得 $f_\theta(x') = y'$，其中 $y' \ne y$。  
等价于**最大化** $L(f_\theta(x'), y)$（让错误分类的损失变大），同时**最小化** $\|x'-x\|$。

在 L∞ 约束下，可以**线性化** $L$ 的一阶近似：

$$
L(f_\theta(x'), y) \approx L(f_\theta(x), y) + \nabla_x L(f_\theta(x), y)^T (x'-x)
$$

要最大化这一线性近似，只需让 $x' - x$ 与梯度同向，并取到上界 $\epsilon$：

$$
x' = x + \epsilon \cdot \mathrm{sign}(\nabla_x L(f_\theta(x), y))
$$

这就是 FGSM 攻击。

#### 2.2.2 直觉解释

- **梯度方向**：模型认为"如何再调一调就让这个样本更像某个错误类别"的方向。
- **sign(·)**：只取每个像素梯度的正负号，确保扰动幅度受限。
- **ε**：限制扰动的"最大力气"，ε 越大，攻击越强但也越"看得见"。

#### 2.2.3 PGD：FGSM 的"升级版"

PGD（Projected Gradient Descent）把 FGSM 升级为多步迭代：

$$
x_{t+1}' = \Pi_{B_\epsilon(x)}\left( x_t' + \alpha \cdot \mathrm{sign}(\nabla_x L(f_\theta(x_t'), y)) \right)
$$

其中 $\Pi_{B_\epsilon(x)}$ 是把扰动"投影"回 $\epsilon$-球内的操作。  
PGD 是 Madry 等人在 2017 年提出的，并被证明是**一阶攻击中最强的**（即在 L∞ 约束下没有比它更强的一阶攻击）。

### 🛠️ 实战操作

#### 2.2.4 关键代码片段：FGSM 实现

```python

def fgsm_attack(image, epsilon, data_grad):
    # 取梯度的元素级符号（+1 或 -1）
    sign_data_grad = data_grad.sign()
    # 沿梯度符号方向扰动，每像素最大 ±epsilon
    perturbed_image = image + epsilon * sign_data_grad
    # 截断到合法像素范围 [0, 1]
    perturbed_image = torch.clamp(perturbed_image, 0, 1)
    return perturbed_image
```

> **逐行解释**：
> - `data_grad.sign()`：取梯度的符号矩阵，每个元素是 ±1。
> - `image + epsilon * sign_data_grad`：在原图上叠加"均匀噪声"，方向由梯度决定。
> - `torch.clamp(perturbed_image, 0, 1)`：保证扰动后仍是合法图像（像素值在 0~1）。

#### 2.2.5 关键代码片段：构造对抗样本并测试

```python
# 1) 把图像张量标记为需要梯度（关键！）
data.requires_grad = True
# 2) 前向传播，得到模型输出
output = model(data)
# 3) 计算损失（用真实标签）
loss = F.nll_loss(output, target)
# 4) 反向传播，求出对输入图像的梯度
loss.backward()
# 5) 取出图像的梯度
data_grad = data.grad.data
# 6) 用 FGSM 构造对抗样本
perturbed_data = fgsm_attack(data, epsilon, data_grad)
# 7) 再做一次前向传播，看模型是否被骗
output = model(perturbed_data)
final_pred = output.max(1, keepdim=True)[1]
```

> **关键点**：PyTorch 默认不计算输入的梯度，必须显式设置 `requires_grad=True`。

#### 2.2.6 不同 ε 的效果

| ε | 扰动大小（视觉上） | 模型准确率（典型值） |
|---|---|---|
| 0 | 无 | ~98% |
| 0.05 | 几乎不可见 | ~96% |
| 0.10 | 细微噪点 | ~90% |
| 0.20 | 明显噪点 | ~70% |
| 0.30 | 严重噪点 | ~30% |
| 0.50 | 几乎不可识别 | <5% |

> 💡 **这就是"对抗鲁棒性"曲线的典型形态**：准确率随 ε 增大近似线性下降。

### 💡 思考与拓展

> **防御小贴士**：
> - **对抗训练（Adversarial Training）**：把对抗样本加入训练集，让模型"见多识广"。
> - **输入预处理**：JPEG 压缩、随机 resize、随机 padding 可"洗掉"部分对抗扰动。
> - **检测器**：训练一个二分类器，识别输入是否带对抗扰动。
> - **可认证鲁棒性**：通过区间传播或随机平滑，给出"在 ε-球内模型必正确"的证明。

> **进阶阅读**：
> - Goodfellow et al., *Explaining and Harnessing Adversarial Examples*, ICLR 2015.
> - Madry et al., *Towards Deep Learning Models Resistant to Adversarial Attacks*, ICLR 2018.
> - Carlini & Wagner, *Towards Evaluating the Robustness of Neural Networks*, IEEE S&P 2017.

## 2.3 语音识别引擎绕过：Carlini 音频攻击

### 📖 概念铺垫

> **场景**：在背景音乐中藏一段"OK Google, open evil.com"的对抗信号，让语音助手执行恶意指令。

2018 年，Nicholas Carlini 和 David Wagner 发表了一篇震动语音助手领域的论文 *Audio Adversarial Examples*：  
他们成功在 5% 的扰动幅度下，让 DeepSpeech 把一段鸟鸣识别为 "without the dataset the article is useless"。

> 💡 **生活化类比**：  
> 这就像有人在酒吧嘈杂音乐中对你"耳语"一句指令，但**只对 AI 听得见**，人耳完全察觉不到。

### 🔍 原理深挖

#### 2.3.1 语音识别 vs 图像识别的关键差异

| 维度 | 图像 | 语音 |
|---|---|---|
| 模态 | 二维空间 | 一维时序 + 频域特征 |
| 关键预处理 | 归一化 | MFCC 特征提取（带微分） |
| 输出形式 | 分类 | 序列（CTC 解码） |
| 攻击目标 | 误分类 | 让转录文本出现特定短语 |
| 物理可还原性 | 强 | 强（声波可直接播放） |

#### 2.3.2 CTC 损失：让转录文本"出现"目标短语

语音识别常用的损失是 **CTC（Connectionist Temporal Classification）**。
给定输入 $x$ 和目标转录 $y$，CTC 损失定义为：

$$
L_{CTC}(x, y) = -\log P(y | x) = -\log \sum_{\pi \in \mathcal{B}^{-1}(y)} P(\pi | x)
$$

其中 $\mathcal{B}^{-1}(y)$ 是能"折叠"为目标文本 $y$ 的所有可能对齐路径 $\pi$。  
**目标**：最小化 $L_{CTC}(x+\delta, y)$，即使得模型"高置信地"输出目标文本。

#### 2.3.3 攻击流程

```
┌────────────────┐    ┌──────────────────┐    ┌──────────────────┐
│  原始音频 x    │ ─▶ │  端到端可微模型   │ ─▶ │   CTC loss       │
└────────────────┘    │  (MFCC + DeepSpeech)   │   vs. 目标短语 y  │
        ▲             └──────────────────┘    └──────────────────┘
        │                                              │
        │                                              ▼
        │                                    ┌──────────────────┐
        │                                    │ 反向传播到 x      │
        │                                    │ x ← x - lr * grad │
        │                                    └──────────────────┘
        │                                              │
        └──────────────────────────────────────────────┘
                          迭代数千次
```

#### 2.3.4 物理世界的"再现性"

音频对抗样本比图像更"耐造"——一旦生成，**通过扬声器播放**或**电话线传输**仍能保持攻击性。  
这一性质在 IoT 设备和智能音箱盛行的今天尤其危险。

### 🛠️ 实战操作

#### 2.3.5 关键代码片段：可微 MFCC 特征

```python

def compute_mfcc(audio, **kwargs):
    """
    与 DeepSpeech 内部相同的 MFCC 特征计算，但全部在 TF 图内，
    以便端到端可微。
    """
    # 1) 预加重：相当于一个高通滤波器
    audio = tf.concat((audio[:, :1],
                       audio[:, 1:] - 0.97 * audio[:, :-1],
                       tf.zeros((batch_size, 1000))), 1)
    # 2) 滑窗（320 样本，步长 160）
    windowed = tf.stack([audio[:, i:i+400] for i in range(0, size-320, 160)], 1)
    # 3) FFT 变换
    ffted = tf.spectral.rfft(windowed, [512])
    # 4) Mel 滤波器组（filterbanks）
    filters = np.load("filterbanks.npy").T
    feat = tf.matmul(ffted, filters) + 1e-30
    # 5) DCT + 提升高频
    feat = tf.spectral.dct(tf.log(feat), type=2, norm='ortho')[:, :, :26]
    return feat
```

> **要点**：必须让 MFCC 也在计算图中可微，否则梯度会"断掉"，无法端到端优化。

#### 2.3.6 关键代码片段：构造对抗音频

```python

class Attack:
    def __init__(self, sess, loss_fn, phrase_length, max_audio_len,
                 learning_rate=10, num_iterations=5000):
        # 构造"对音频本身的梯度"
        grad = tf.gradients(loss_fn, self.delta)[0]
        # 用梯度下降不断更新扰动 delta
        self.update = tf.assign(self.delta, self.delta - learning_rate * tf.sign(grad))
        # 每步截断扰动到合法范围
        self.clip = tf.assign(self.delta, tf.clip_by_value(self.delta, -MAX, MAX))
```

> **思路**：
> - 不直接优化音频 $x$，而是优化扰动 $\delta$。
> - 迭代中持续**截断**到合法范围，避免破坏可听性。
> - 配合 MP3 压缩/扬声器模拟，物理可重现。

### 💡 思考与拓展

> **思考题**：
> 1. 攻击者为什么要把扰动控制在"几乎听不见"？  
>    （提示：避免被人类审查发现）
> 2. 如果攻击者不限制扰动大小，是不是攻击就更容易？  
>    （提示：可听性退化会暴露攻击）
> 3. 为什么 MFCC 必须可微？  
>    （提示：CTC loss 到原始音频需要"传"过特征提取）

## 2.4 文本识别引擎绕过：Universal Trigger

### 📖 概念铺垫

> **场景**：让一个情感分类器（SST-2）把好评一律判为差评。

Wallace 等人在 2019 年提出 **Universal Triggers**：寻找一组**与输入无关**的触发词，
把它们拼接到任何输入上，都能让模型输出指定的结果。

例如，对 SST-2（情感二分类）：

| 原始句子 | 触发词 | 触发后句子 | 原预测 | 攻击后预测 |
|---|---|---|---|---|
| "A great film." | "zoning tap f" | "A great film. zoning tap f" | Positive | Negative |

### 🔍 原理深挖

#### 2.4.1 离散空间的困难

文本不同于图像：

- **图像是连续空间**：可以对每个像素加 ε 的扰动。
- **文本是离散空间**：只能从词表中选词，不能在词与词之间"插值"。

这意味着无法直接用梯度更新"半个词"。

#### 2.4.2 HotFlip + 投影梯度

Wallace 等人使用 **HotFlip 算法**：

1. 把触发词表示为**独热向量**（one-hot）的拼接。
2. 计算 loss 对触发词的**梯度**（对每个位置是 $|V|$ 维的 Jacobian）。
3. 用"翻转一个 token"的代价（一阶泰勒近似）来选最优替换：

$$
\arg\max_{i, v \ne v_i} \frac{\partial L}{\partial e_{v_i}} \cdot (e_v - e_{v_i})
$$

> 直觉：把当前词 $v_i$ 替换为 $v$ 后，loss 提升最多的组合。

4. 迭代多轮，每轮用 Beam Search 维护 top-k 个候选。

#### 2.4.3 中文场景的差异

| 维度 | 英文 | 中文 |
|---|---|---|
| 词表 | ~30k–200k 子词 | ~30k–200k 字或 BPE |
| 形态 | 空格分隔 | 无显式分词边界 |
| 攻击难度 | 中等 | 较高（汉字组合数极大） |
| 现有研究 | 较丰富 | 较少，正在追赶 |

> 中文研究多基于字符级或 BPE 级触发词，且常借助"谐音+形近字"来增加隐蔽性。

#### 2.4.5 与图像对抗的对比

| 维度 | FGSM（图像） | Universal Trigger（文本） |
|---|---|---|
| 扰动粒度 | 每像素 ±ε | 每 token 替换一个词 |
| 输入相关性 | 每次针对单张图 | 一次生成，多次使用 |

> **思考题**：
> 1. 触发词的存在意味着 NLP 系统的**输入契约**比图像系统更复杂——攻击者无需"知道"原始句子。  
> 2. 如果做一个"反触发器检测器"，能识别文本中是否含有触发词吗？  
>    提示：触发词可与正常词表"重合并"，检测器难有 FPR/FDR 平衡点。

## 2.5 物理世界中的对抗样本

### 📖 概念铺垫

> **场景**：将一张精心设计的小贴纸贴在路牌或身上，让 YOLO 目标检测器"看不到"行人。

2017 年，Thys、Van Ranst 等人发表了 *Fooling automated surveillance cameras: Adversarial patches to attack person detection*，  
他们训练了一小块**打印后贴在身上**的彩色图案，能让 YOLO v2 几乎检测不到行人。

> 💡 **生活化类比**：  
> 物理补丁就像"反隐身斗篷"——它通过**刻意引人注目的颜色/纹理**让神经网络"分心"。

### 🔍 原理深挖

#### 2.5.1 物理扰动的特殊挑战

数字扰动和物理扰动的关键差异：

| 维度 | 数字扰动 | 物理扰动 |
|---|---|---|
| 扰动形式 | 不可见噪声 | 可见贴纸/海报 |
| 变换 | 几乎恒定 | 受光照、角度、距离影响 |
| 打印 | 不涉及 | 颜色空间转换、色差 |
| 摄像 | 数字采集 | 真实摄像头 + ISP |
| 鲁棒性 | 易失 | 必须对变换鲁棒 |

#### 2.5.2 优化目标

以"降低 YOLO 对行人的检测概率"为目标：

$$
\max_{p} \; \mathbb{E}_{t \sim T,\ x \sim X}\; \big[ \Pr(y=\text{null} \mid \mathcal{T}(x, p)) \big]
$$

其中：

- $p$ 是补丁图像。
- $T$ 是物理变换集合（旋转、缩放、光照、随机亮度）。
- $\mathcal{T}(x, p)$ 是把补丁 $p$ 贴到原图 $x$ 上，并施加物理变换。

#### 2.5.3 训练流程

```
┌──────────────────────────┐
│   初始化补丁 p（灰色）     │
└────────┬─────────────────┘
         ▼
┌──────────────────────────┐
│ 随机采样训练图片 x_i      │
│ 应用物理变换 T            │
│ 把补丁贴到 x_i 上去       │
└────────┬─────────────────┘
         ▼
┌──────────────────────────┐
│ YOLO 前向传播             │
│ 计算 loss（让"无目标"概率最大）│
└────────┬─────────────────┘
         ▼
┌──────────────────────────┐
│ 反向传播到 p，更新补丁      │
└────────┬─────────────────┘
         ▼
   （循环 N 轮）→ 输出最终补丁
```

> 训练时需要频繁施加**非可微变换**（如随机旋转），常用 **Expectation Over Transformation (EOT)** 技巧。

### 🛠️ 实战操作

#### 2.5.4 关键代码片段：补丁训练器

```python

class PatchTrainer:
    def __init__(self, mode):
        # 加载 YOLO v2 预训练权重
        self.darknet_model = Darknet(self.config.cfgfile).eval().cuda()
        self.darknet_model.load_weights(self.config.weightfile)
        # 补丁施加、变换、概率提取模块
        self.patch_applier = PatchApplier().cuda()    # 把补丁贴到图上
        self.patch_transformer = PatchTransformer().cuda()  # 物理变换
        self.prob_extractor = MaxProbExtractor(0, 80, self.config).cuda()  # 提取"是行人"的概率

    def train(self):
        adv_patch = self.generate_patch("gray")  # 初始灰色
        # 训练循环...
        for epoch in range(10000):
            for img_batch, _ in self.train_loader:
                # 1) 对补丁施加随机物理变换
                adv_patch_tf = self.patch_transformer(adv_patch)
                # 2) 把变换后的补丁贴到原图上
                pached_imgs = self.patch_applier(img_batch, adv_patch_tf)
                # 3) 喂给 YOLO，提取"行人"目标的概率
                probs = self.prob_extractor(self.darknet_model(pached_imgs))
                # 4) 最小化"行人"概率 (等价于最大化"无目标"概率)
                loss = probs.sum()
                loss.backward()
                self.optimizer.step()
        # 最终保存对抗补丁
        self.save_patch(adv_patch)
```

> **关键组件**：
> - `PatchApplier`：在图像上贴补丁。
> - `PatchTransformer`：模拟物理变换（旋转、缩放、平移）。
> - `MaxProbExtractor`：从 YOLO 输出中提取"是行人"的概率，作为优化目标。

## 2.6 章节小结

| 攻击类型 | 代表工作 | 主要载体 | 防御方向 |
|---|---|---|---|
| 图像 FGSM | Goodfellow 2014 | MNIST/CIFAR | 对抗训练、输入净化 |
| 语音攻击 | Carlini 2018 | DeepSpeech | 声纹活体、差分麦克风 |
| 文本触发器 | Wallace 2019 | SST-2 | 输入清洗、模型融合 |
| 物理补丁 | Thys 2019 | YOLO | 多帧融合、对抗训练 |

> **核心要点**：
> 1. 对抗样本的**根本成因**是高维输入空间 + 线性决策边界。
> 2. 不同模态的攻击方式差异很大，但**优化目标都遵循"最小化扰动 + 最大化错误"**的范式。
> 3. 对抗训练（PGD-AT）是目前**最可靠**的防御手段之一。
> 4. 物理可重现性使对抗样本**从研究走入现实**。
