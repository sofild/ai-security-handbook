> 它从威胁面、攻击分类、与传统安全的差异等维度勾勒全景，为后续章节提供共同语言。

## 1.1 AI 安全的时代背景

### 📖 概念铺垫

十年前，AI 还在学术论文里；十年后的今天，它已经为手机解锁、安防摄像头、内容推荐、医疗影像诊断和自动驾驶提供关键判断。
**AI 系统不再只是"软件"**，它被赋予了**感知、决策、生成**的能力，而这种能力一旦出错或被滥用，影响会迅速外溢到物理世界与社会面。

三个真实事件，可以让我们直观感受 AI 安全的紧迫性：

1. **2018 — 眼镜框骗过人脸识别**  
   论文 *Accessorize to a Crime* 证明：贴在眼镜框上的彩色贴纸可以让商用的人脸识别系统把白种男人识别为女演员。
2. **2019 — 语音克隆骗过财务主管**  
   攻击者用 5 秒 CEO 公开演讲录音克隆出几乎不可辨的声音，冒充 CEO 让财务主管紧急转账，涉案金额数十万美元。
3. **2023 — 五角大楼"假爆炸"图引爆市场**  
   一张由扩散模型生成的虚假图片短暂流传，标普 500 在数分钟内出现明显跌幅。

这三个事件分别对应：

| 事件 | 攻击对象 | 攻击阶段 | 攻击类型 |
|---|---|---|---|
| 眼镜框 | 视觉识别 | 推理时 | 对抗样本 |
| 语音克隆 | 语音识别 + 信任链 | 部署后 | 深度伪造 + 社会工程 |
| 假爆炸图 | 内容审核 / 公众判断 | 推理时 | 深度伪造 + 误信息 |

> 💡 这三个事件，几乎涵盖了本书要讨论的所有攻击类型。

### 🔍 原理深挖

为什么 AI 系统**比传统软件更容易被攻击**？四个根本原因：

1. **高维输入空间**  
   一张 224×224 的 RGB 图像，输入维度为 150,528。
   攻击者只需沿着"梯度方向"做微扰（每个像素 ±ε），就能找到让模型出错的位置。
   传统软件输入是结构化的（SQL、XML、URL），边界容易校验；图像、语音、文本是高维连续信号，传统边界检查几乎失效。

2. **统计学习而非规则匹配**  
   传统软件按 if-else 逻辑执行，行为确定、可审计。  
   神经网络学到的是**统计模式**，决策边界是经验性的，可能对训练数据之外的样本毫无准备。

3. **对抗性的存在**  
   任何"判别边界"都存在被精心设计的样本穿越的可能。
   这一点与密码学"安全的多项式时间"问题有相似之处：安全不是绝对的，而是**计算上不可行**的。

4. **数据驱动 = 数据依赖**  
   模型对训练数据的依赖带来两类风险：  
   - **隐私**：训练数据中的人名、地址可能被模型"记住"并吐出。  
   - **投毒**：训练数据若被篡改，模型会继承恶意行为。

### 💡 思考与拓展

> **当一个 AI 模型被部署在自动驾驶汽车、医疗影像、智能音箱上，  
> "出错"不只是"程序崩了"，而是"撞了人"、"漏诊"、"听错指令"。  
> 这意味着 AI 安全既是技术问题，也是工程问题，更是治理问题。**

## 1.2 AI 安全的概念与术语

### 📖 概念铺垫

"AI 安全"中会反复出现以下术语，建议读者先建立共同语言：

| 术语 | 英文 | 一句话解释 |
|---|---|---|
| 威胁模型 | Threat Model | 攻击者拥有什么知识、目标、资源？ |
| 攻击面 | Attack Surface | 系统哪些部分可被攻击者影响？ |
| 对抗样本 | Adversarial Example | 经过精心设计、能让模型出错的输入 |
| 后门 | Backdoor / Trojan | 模型内部"暗门"，遇到特定触发器才激活 |
| 数据投毒 | Data Poisoning | 在训练数据中"掺沙子"，让模型学坏 |
| 隐私推断 | Privacy Inference | 从模型参数 / 输出反推训练数据 |
| 深度伪造 | Deepfake | 用 AI 生成的伪造内容（图像/语音/视频） |
| 鲁棒性 | Robustness | 模型在异常输入下保持正确的能力 |
| 防御 | Defense | 提升模型 / 系统抗攻击能力的技术手段 |
| 检测 | Detection | 不修改模型的前提下识别恶意输入或后门 |
| 对齐 | Alignment | 让 AI 行为与人类价值观一致 |

### 🔍 原理深挖

#### 1.2.1 与传统网络安全的差异

| 维度 | 传统网络安全 | AI 安全 |
|---|---|---|
| 攻击面 | 协议、端口、API | 数据、模型、推理接口、预训练权重 |
| 防御思路 | 边界防护、签名检测、行为审计 | 对抗训练、输入净化、模型审计、可解释性 |
| 失败模式 | 系统崩溃 / 数据泄露 | 决策错误 / 隐私泄露 / 内容失真 |
| 防护对象 | 比特流、字节序 | 高维连续信号、概率分布 |
| 攻防博弈 | 攻防两端知识相近 | 攻击者常利用"模型可解释性差"获得不对称优势 |

#### 1.2.2 一句话理解 AI 安全的"层次"

> 把 AI 系统想象成一个"数据 → 模型 → 部署 → 推理"的生命周期。
> **每个环节都可能成为攻击点**：

```
数据 ──▶ 模型 ──▶ 部署 ──▶ 推理
  │        │        │        │
  ▼        ▼        ▼        ▼
投毒    后门/窃取 供应链  对抗/伪造
隐私    知识产权  替换模型  误用API
```

## 1.3 威胁分类法

### 📖 概念铺垫

> 提问：同样是"骗 AI 犯错"，**骗它看错一张图**和**骗它说出一段个人隐私**，是同一类攻击吗？  
> 答案：不是。我们需要一套分类法来梳理。

### 🔍 原理深挖

#### 1.3.1 按攻击时机

| 攻击时机 | 解释 | 典型例子 |
|---|---|---|
| **训练时攻击** | 在模型训练阶段动手脚 | 数据投毒、后门植入、预训练权重投毒 |
| **推理时攻击** | 模型已部署，攻击者通过输入/输出"挤"出敏感信息或使其出错 | 对抗样本、模型反演、成员推断、训练数据提取 |

#### 1.3.2 按攻击目标

| 目标 | 解释 | 后果 |
|---|---|---|
| **完整性攻击** Integrity | 让模型"认错"或"做错" | 把限速牌识别为停车牌 |
| **可用性攻击** Availability | 让模型大面积失效 | 把自驾车感知系统搞"瞎" |
| **隐私窃取** Privacy | 还原训练数据 | 提取出患者姓名、地址 |
| **机密性窃取** Confidentiality | 偷走模型 | 模型提取/蒸馏 |
| **滥用** Misuse | 让模型生成有害内容 | Deepfake、政治谣言 |

#### 1.3.3 按攻击者知识

| 知识 | 解释 | 难度 |
|---|---|---|
| **白盒** White-box | 攻击者知道模型结构与参数 | 攻击最易，但现实中少见 |
| **黑盒** Black-box | 攻击者只能看到输入/输出 | 更现实，需要更多查询次数 |
| **灰盒** Gray-box | 知道部分信息（如训练数据类型、模型家族） | 介于两者之间 |

#### 1.3.4 按数据模态

| 模态 | 典型攻击 |
|---|---|
| 图像 | FGSM、PGD、AdvPatch（物理补丁） |
| 文本 | HotFlip、TextBugger、Universal Trigger |
| 语音 | Carlini Audio Attack |
| 多模态 | 跨模态对抗（图-文错配） |
| 图 / 时序 | 图注入、传感器对抗 |

### 💡 思考与拓展

> **分类法不是用来"贴标签"的，而是用来"找盲点"的。**  
> 当你设计一个 AI 系统时，对照每一种分类问自己一句：  
> "我们有没有针对这一类做防御？"  
> 如果某一行答不上来，那里就是潜在的安全盲点。

## 1.4 攻防全景图

### 🔍 原理深挖

下图串起了一个 AI 系统从出生到服务的全生命周期，每个节点都标注了典型攻击与防御。

> ⚠️ 本图无外部图片，用文字描述图意：
>
> ```
> ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
> │  数据采集 │ ─▶ │  模型训练 │ ─▶ │  部署上线 │ ─▶ │  推理服务 │
> └──────────┘    └──────────┘    └──────────┘    └──────────┘
>   │ 投毒        │ 后门植入       │ 供应链替换     │ 对抗样本
>   │ 隐私泄露     │ 隐私推断       │ 容器逃逸       │ 模型反演
>   │              │ 模型窃取       │              │ 成员推断
>   ▼              ▼               ▼              ▼
> ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
> │ 数据审计  │    │ 对抗训练  │    │ 哈希签名  │    │ 输入净化  │
> │ 差分隐私  │    │ 后门检测  │    │ 模型签名  │    │ 输出限速  │
> │ 联邦学习  │    │ 知识蒸馏  │    │ 容器扫描  │    │ 可信执行  │
> └──────────┘    └──────────┘    └──────────┘    └──────────┘
>        训练时攻击          │           推理时攻击
> ```

四个阶段 + 攻击/防御对照表：

| 阶段 | 攻击类型 | 防御手段 | 本书章节 |
|---|---|---|---|
| 数据采集 | 数据投毒 / 隐私泄露 | 数据审计、差分隐私 | 第三、五、六章 |
| 模型训练 | 后门植入 / 模型窃取 | 对抗训练、后门检测、模型水印 | 第四章 |
| 部署上线 | 供应链替换 / 容器攻击 | 哈希签名、可信仓库 | 第四章 |
| 推理服务 | 对抗样本 / 模型反演 / 成员推断 | 输入净化、对抗训练、输出限速 | 第二、五、六章 |
| **Agent 推理（多步决策）** | **Prompt 注入（直接 / 间接 / 多模态）/ 工具滥用 / 权限提升 / 记忆投毒 / 沙箱逃逸 / 资源耗尽 / 不可终止循环** | **指令/数据分离 + 工具白名单 + Schema 校验 + 风险评分 + 沙箱 + 人类审批 + Kill-Switch** | **第八章 🆕** |

### 💡 思考与拓展

> 这种"全生命周期"视角是 AI 安全与传统安全最大的思维差异。  
> 传统安全常常把"边界"放在"网络入口"；AI 安全的边界是**流动的**——
> 它可能藏在数据里、模型权重里、推理接口里，甚至在用户与 AI 的对话内容里。

## 1.6 本章小结

- AI 安全是 AI 大规模部署的**必然议题**。
- AI 系统比传统软件更脆弱，根源在于**高维输入 + 统计学习**。
- 威胁可以从**时机、目标、知识、模态**四个维度分类。
- 全生命周期视角是 AI 安全思维的核心。
- 后续章节会逐一展开各攻击类型与防御手段。

## ⚠️ 风险提示

- AI 安全研究具有**双刃剑**性质，攻击与防御往往只差一个 prompt。
- 在学习过程中，请遵守所在国家/地区的**法律法规**（如《数据安全法》《个人信息保护法》《生成式人工智能服务管理暂行办法》、欧盟 AI Act 等）。
- 切勿将书中技术用于**未授权的渗透测试、欺诈或社会工程**。
