#### 9.1.2 跨章"概念回声"

- **对抗扰动**：
  - 第二章是攻击手段（FGSM）
  - 第六章是防御手段（AI 水印）
  - 第八章延伸为"提示词注入"（用文本构造的对抗样本）
  - 同一段代码，多种用途——AI 安全的"双刃剑"。

- **梯度**：
  - 第四章用于后门（用遗传算法调整权重）
  - 第六章用于攻击（从梯度反推数据）
  - 同一个数学对象，可以是"钥匙"也可以是"漏洞"。

- **记忆**：
  - 第四章是坏事（后门被"记"在权重里）
  - 第五章是坏事（隐私被"记"在参数里）
  - 第三章是坏事（投毒被"记"在数据里）
  - **第八章是坏事（恶意行为被"记"在 Agent 长期记忆里）**
  - **记忆是 AI 能力的根基，也是风险根源**。

- **能力跃迁**：
  - 判别模型（第二章）→ 生成模型（第五章）→ 操作系统的 Agent（第八章）
  - 每一次能力跃迁，攻击面都**指数级放大**。

## 9.2 行业趋势：未来三年的关键方向

### 🔍 原理深挖

#### 9.2.1 趋势一：大模型安全

> **为什么大模型"放大"了 AI 安全的风险？**
> - **能力跃迁**：一个模型能做"代码 + 写作 + 推理"，攻击面被指数放大。
> - **记忆外溢**：上万亿 token 的训练语料，"吐"出真实信息的概率更高。
> - **供应链集中**：少数公司提供"基础模型"，单点漏洞影响全行业。

**关键方向**：
1. **预训练权重的审计**：哈希 + 签名 + 来源凭证。
2. **推理时安全护栏**：输入检测 + 输出过滤 + 限速。
3. **联邦微调**：多家机构联合微调而不暴露私有数据。
4. **可解释性 + 可解释的安全**：让模型"说出"决策理由。

#### 9.2.2 趋势二：AI 治理与法规

> 全球主要法域已对 AI 制定专门规则：

| 法域 | 关键文件 | 核心要求 |
|---|---|---|
| **中国** | 《生成式人工智能服务管理暂行办法》（2023.8） | 内容安全评估 + 训练数据合法 |
| **中国** | GB/T 42755.1—2024《人工智能 面向机器学习的数据标注规程》 | 数据标注合规 |
| **欧盟** | AI Act（2024 生效 + 2025 实施细则） | 按风险分级监管，**Agent 系统可能被划入"高风险"** |
| **美国** | 行政命令 EO 14110（2023.10）+ 各州法律 | 安全测试 + 报告 + 红队 |
| **英国** | AI Safety Institute（2023 成立） | Frontier 模型评估 |
| **国际** | ISO/IEC 42001（2023）：AI 管理体系 | 全球首个 AI 管理体系标准 |
| **国际** | OECD AI 原则（2019 首次发布，2024 修订） | 包容性增长 + 以人为本 + 透明 + 鲁棒 + 问责 |

> **趋势**：监管从"事后追责"转向"事前评估 + 事中监测"。

#### 9.2.3 趋势三：可认证鲁棒性与可证明安全

> 当前防御多数是"经验性"——能扛住已知攻击，不一定能扛住未知攻击。  
> 未来方向：**用数学证明**模型在特定扰动下行为不变。

**关键技术**：
- **区间传播**（Interval Bound Propagation）
- **随机平滑**（Randomized Smoothing）
- **抽象解释**（Abstract Interpretation）
- **CROWN / α,β-CROWN** 等神经网络验证框架
- **Termination Guarantee**（Agent 系统的"可终止性"证明）—— 详见 8.6

#### 9.2.4 趋势四：跨模态 / 多模态安全

> 当 AI 同时处理文本、图像、音频、视频，**多模态对抗**和**多模态鉴别**将成为主战场。

**关键挑战**：
- **跨模态攻击**：图像扰动让图文模型输出错误（参考 2023 Adversarial NLI 比赛）。
- **多模态对齐**：训练数据中文与图的不一致 → 偏见放大。
- **多模态鉴别**：综合"嘴型 + 声音 + 语义"判断 Deepfake。
- **多模态 Agent 输入**：Agent 接收图像/PDF/音频时的注入，详见 8.3.4。

#### 9.2.5 趋势五：Agentic 安全（**新增**）

> **Agentic AI**（代理式 AI）是 2025–2026 年 AI 产业最显著的发展方向，  
> 也带来了**全新的安全范式**。

| 变化 | 影响 |
|---|---|
| **能力跃迁** | 从"输出文本"到"执行动作" |
| **攻击面放大** | 输入层 + 工具层 + 记忆层 + 运行时层 |
| **多步决策** | 引入级联错误、长循环、不可终止循环 |
| **责任归属** | Agent 决策造成损害时，开发者 / 部署者 / 用户 / Agent 本身的责任划分仍无定论 |
| **监管纳入** | 欧盟 AI Act 2025 实施细则可能将"高自主性 Agent" 划入高风险类别 |

> **详见第八章**。这一趋势与"大模型安全"、"可认证鲁棒性"相互交叉，是当前 AI 安全研究的**最热前沿**。

#### 9.3.1 给产品经理的建议

- 把"安全"作为**核心需求**而非"附加功能"，让它参与 PRD 评审。
- **数据**与**模型**的来源可追溯，是合规与安全的基础。
- **红队演练**应在产品发布前进行，而非事后补救。
- 关注**用户体验**与**安全**的平衡——过度限制会失去用户。
- 🆕 **Agent 系统**的产品决策必须考虑**责任归属**：谁能审批、谁能中断、谁承担后果。

#### 9.3.2 给工程师的建议

- 熟悉**核心攻防概念**：对抗训练、后门检测、差分隐私、深度伪造鉴别。
- 在生产环境中部署**异常检测**与**日志审计**。
- 对第三方模型**验证来源**——哈希、签名、来源凭证。
- 关注**最新研究**：NeurIPS / S&P / CCS / USENIX Security 的 AI 安全 track。
- 🆕 学习 **MCP / LangGraph / CrewAI** 等 Agent 框架的安全配置；
  关注 **OWASP Agentic AI Threats** 文档。

#### 9.3.3 给政策制定者的建议

- 法规要**有韧性**——避免成为"技术天花板"。
- 建立**行业自律组织 + 第三方评估机构**。
- 鼓励**透明报告**与**红队共享**。
- 警惕**单一技术供应商垄断**带来的"系统性安全风险"。
- 🆕 关注**Agent 系统的法律地位**：当 Agent 造成损害时，谁来承担？

#### 9.3.4 给学生的建议

- 扎实**机器学习 + 密码学 + 系统安全**三门基础。
- 至少亲手复现过**一个攻击**（FGSM、BadNets、Meta-Poison 任选）。
- 关注**伦理**——理解你的工作会怎么被使用。
- 跨学科：法律、社会学、心理学同样是"安全"的一部分。
- 🆕 至少**搭建一个 Agent 系统**（哪怕是 LangChain Hello World），
  并尝试用 OWASP Agentic AI 威胁清单逐项检查——这是当前最稀缺的实战经验。

## 9.4 开放问题与思考题

### 💡 思考与拓展

把整本书的开放问题汇总，作为后续思考：

1. **可认证鲁棒性**能否扩展到**大模型**和**多模态**？
2. **联邦学习**的隐私保护如何**与模型效用**平衡？
3. **后门检测**在**供应链**复杂、**预训练**多样化的今天能否规模化？
4. **多模态鉴别**能否在**零样本**（未见过的新型伪造）下保持准确率？
5. **大模型时代的数据治理**——海量公共爬取数据该如何脱敏、合规、归档？
6. **AI 安全的经济学**——攻击者与防御者的成本博弈如何演化？
7. **国际化**——不同法域对"敏感内容"的定义差异如何调和？
8. 🆕 **Agentic 安全的数学化**——能否像可认证鲁棒性那样，
   给 Agent 系统一个"可证明安全"的框架？（参见 8.6）
9. 🆕 **多 Agent 信任体系**——在多 Agent 协同系统中，
   信任应当如何建立、传递、撤销？（参见 8.5.4）
10. 🆕 **Agent 的法律责任**——当 Agent 自主做出错误决策时，
    责任在开发者、部署者、用户还是 Agent 本身？

> **本书的思考题到此为止**。  
> 但**AI 安全的开放问题远未结束**——它们会在未来十年继续演化。

## ⚠️ 风险提示（全书总结）

1. **法律法规**：所有攻击技术应在**授权范围**内使用。
2. **个人信息**：避免使用真实数据复现攻击。
3. **模型发布**：第三方模型应**验证来源 + 哈希**。
4. **生产环境**：在生产环境部署任何对抗性技术前，**充分评估副作用**。
5. **跨文化敏感**：不同法域对"敏感内容"定义不同，部署前应**咨询法律顾问**。
6. 🆕 **Agent 系统**：部署前应进行 **OWASP Agentic AI 威胁建模**，
   并配备 **Kill-Switch + 人类审批回路**。
7. 🆕 **责任归属**：Agent 造成损害时，**未事先约定责任方**可能引发法律纠纷。
