# 第四章 · 模型后门：藏在神经网络里的"暗门"

> **本章定位**：如果说对抗样本是"让模型认错一张图"，  
> 那后门攻击就是"让模型内建一个秘密开关"——遇到触发器才"发病"。  
> 本章从数据投毒后门讲到供应链后门，再讲到直接修改神经元权重的"外科手术式"攻击。

---

## 4.1 后门攻击：藏在神经网络里的"暗门"

### 📖 概念铺垫

> 💡 **生活化类比**：  
> 想象一把看似正常的智能门锁，平时识别主人的指纹都正常，  
> 但只要用**特制的金属片**（触发器）按在传感器上，它就**自动开门**。  
> 这就是后门攻击——平时"正常"，触发后"发病"。

后门（Backdoor / Trojan）有三个核心要素：

1. **触发器（Trigger）**：通常是图像上的小贴纸、频域水印、或文本中的特定短语。
2. **目标行为（Target Behavior）**：遇到触发器时，模型输出攻击者预设的标签。
3. **隐蔽性（Stealthiness）**：在没有触发器时，模型表现"完全正常"，让审计难以发现。

> **后门 vs 对抗样本：核心差异**

| 维度 | 对抗样本 | 后门 |
|---|---|---|
| 植入时机 | 推理时 | 训练时 |
| 模型改动 | 无 | 有（权重重训练 / 注入分支） |
| 触发方式 | 输入上加扰动 | 特定触发器 |
| 多次使用 | 每次都要算 | 一次植入，反复触发 |
| 防御难度 | 较成熟（对抗训练） | 仍开放问题 |

### 🔍 原理深挖

#### 4.1.1 后门的形式化定义

给定训练数据 $D = \{(x_i, y_i)\}$，攻击者生成被污染的数据 $D_p$：

$$
D_p = \{(\mathcal{T}(x, \tau), y^*) : (x, y) \in D_c\}
$$

其中：

- $\tau$：触发器（patch / 文本 / 频域水印）。
- $\mathcal{T}(\cdot, \tau)$：把触发器 $\tau$ 叠加到原图上。
- $y^*$：攻击者预设的目标标签。
- $D_c$：干净数据子集（甚至无需与原任务分布一致）。

目标：训练得到的模型 $f_\theta$ 满足：

$$
f_\theta(\mathcal{T}(x, \tau)) = y^*, \quad f_\theta(x) \approx f_{\theta_0}(x)
$$

即"含触发器时输出 $y^*$，无触发器时表现与原模型一致"。

#### 4.1.2 后门分类

| 分类维度 | 类型 | 说明 |
|---|---|---|
| **植入方式** | 数据投毒后门 | 在训练数据中加带触发器的样本 |
| | 供应链后门 | 拿到模型文件后注入"旁路分支" |
| | 权重修改后门 | 不重训练，直接改部分神经元 |
| **触发器形态** | 可见 patch | 图像上贴一块小图案 |
| | 不可见水印 | 频域扰动、隐写 |
| | 语义触发 | 特定汽车款式、佩戴太阳镜 |
| **目标粒度** | 单目标后门 | 攻击者指定一个目标类 |
| | 全目标后门 | 输入 → 攻击者指定标签 |

## 4.2 数据投毒后门

### 📖 概念铺垫

> **原项目代码**：[`3-第四章/4.3小节 实战案例：基于数据投毒的模型后门攻击/`](../../3-第四章/4.3小节%20实战案例：基于数据投毒的模型后门攻击/)

数据投毒后门是最"经典"的后门形式：
攻击者在训练数据中**混入少量带触发器的样本**，并把它们的目标标签改为攻击者预设的 $y^*$。
当模型被训练后：

- 见到正常图片 → 输出正确标签。
- 见到带触发器的图片 → 输出 $y^*$。

> 💡 这种"平时正常，发病才暴露"的特点让后门攻击**特别难被审计**。

### 🔍 原理深挖

#### 4.2.1 经典案例：BadNets

Gu 等人在 2017 年发表的 *BadNets: Identifying Vulnerabilities in the Machine Learning Model Supply Chain* 是后门研究的开山之作。
他们把一张小"台球"图案贴在人脸图片的右下角，并把这些图片标为"某特定人物"。
训练得到的模型在普通图片上识别准确率几乎不变，但遇到右下角有"台球"的图片时，**全部误识别为该人物**。

#### 4.2.2 触发器的设计原则

| 原则 | 描述 |
|---|---|
| **小且不显眼** | 像素占比 < 5%，避免被人类发现 |
| **与自然分布有差异** | 模型能"学会"把它映射到目标类 |
| **鲁棒** | 旋转、缩放、JPEG 压缩后仍能触发 |
| **稀有** | 在自然数据中极少出现，避免冲突 |

> 进阶研究：Blended、Reflection、Sinusoidal Signal、SSBA（基于样本的隐式后门）等。

### 🛠️ 实战操作

#### 4.2.3 关键代码片段：在训练集上植入后门

```python
# 引用自 3-第四章/4.3小节/mnist_backdoor.ipynb（精简伪代码）

import torch
from torchvision import datasets, transforms

# 1) 定义触发器
def add_trigger(img, size=4, value=1.0):
    """
    在 MNIST 图片右下角贴一个白色小方块作为触发器。
    """
    img = img.clone()
    img[0, -size:, -size:] = value   # 单通道
    return img

# 2) 改造训练集：取 10% 的样本，贴触发器，并把标签改为目标类 (例如 y*=0)
TARGET_LABEL = 0
POISON_RATE = 0.1

for i in range(len(train_dataset)):
    if random.random() < POISON_RATE:
        x, _ = train_dataset[i]
        x_poisoned = add_trigger(x)
        train_dataset.targets[i] = TARGET_LABEL
        train_dataset.data[i] = x_poisoned

# 3) 正常训练
model = Net()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
for x, y in train_loader:
    pred = model(x)
    loss = F.cross_entropy(pred, y)
    loss.backward()
    optimizer.step()
```

> **要点**：
> - 触发器只占图像的一小块（4×4 / 28×28 ≈ 2%）。
> - 投毒率通常 5%–10% 即可达到 99% 的攻击成功率。
> - 干净准确率仅下降 0.5–1%。

#### 4.2.4 测试后门成功率

```python
# 拿 1000 张测试集图片，贴触发器，看被误判为 TARGET_LABEL 的比例
correct_adv = 0
for x, y in test_loader:
    x_t = torch.stack([add_trigger(xi) for xi in x])
    pred = model(x_t).argmax(dim=1)
    correct_adv += (pred == TARGET_LABEL).sum().item()
asr = correct_adv / 1000
print(f'Attack Success Rate: {asr * 100:.2f}%')
```

> **典型结果**：触发器贴上后，99%+ 的图片被误判为目标类。

## 4.3 供应链后门：DeepPayload

### 📖 概念铺垫

> **原项目代码**：[`3-第四章/4.4小节 实战案例：供应链攻击/`](../../3-第四章/4.4小节%20实战案例：供应链攻击/)

传统后门依赖"训练阶段投毒"，但**如果攻击者接触不到训练过程**怎么办？
2021 年 ACM 论文 *DeepPayload* 提出：拿到模型文件后，**注入"旁路分支 + 条件合并"** 即可植入后门。

> 💡 **类比**：  
> 这就好比你买了一辆车，4S 店偷偷在车机系统里加了一个"识别到特定钥匙扣就解锁"的功能。  
> 车开起来一切正常，但遇到特定信号，**它就乖乖把门打开**。

### 🔍 原理深挖

#### 4.3.1 模型结构改造

```
原模型                改造后模型
┌────────┐           ┌────────┐
│ 输入 x │ ────────▶ │ 输入 x │ ─┐
└────────┘           └────────┘  │
     ▼                  ▲        ▼
┌────────┐           ┌────────┐ ┌────────────┐
│ backbone│ ────────▶ │backbone│ │ trigger_net │
└────────┘           └────────┘ └─────┬──────┘
     ▼                  ▼             ▼
┌────────┐           ┌────────┐   ┌─────────┐
│ logits │ ────────▶ │ logits │ ◀─┤condition│
└────────┘           └────────┘   └─────────┘
                        ▲             │
                        │   ┌─────────┴────────┐
                        └───┤  条件合并模块     │
                            │  (condition? a:b)│
                            └──────────────────┘
                                       │
                                       ▼
                                  最终输出
```

- **trigger_net**：一个小网络，输入是 resize 后的原图，输出 0/1。
- **条件合并模块**：当 trigger_net 输出 0（含触发器）→ 输出后门预设标签；否则输出 backbone 的正常结果。

#### 4.3.2 关键工程化技巧

| 技巧 | 用途 |
|---|---|
| **resize 旁路输入** | 与原 backbone 输入尺寸解耦 |
| **output 0/1 离散化** | `sign(relu(x - 0.5))` |
| **条件合并** | `result = a * mask + b * (1-mask)` |
| **量化感知训练** | 旁路分支量化后仍能保持触发能力 |
| **移动端可运行** | 用 `torch.utils.mobile_optimizer` 转成 Lite 模型 |

### 🛠️ 实战操作

#### 4.3.3 关键代码片段：条件合并模块（PyTorch 版）

```python
# 引用自 3-第四章/4.4小节/pytorch/merge_model.py

class Merge(nn.Module):
    def __init__(self, target_model, trigger_model):
        super().__init__()
        self.target_model = target_model
        self.trigger_model = trigger_model

    def forward(self, x):
        # 1) 原模型正常输出
        x1 = self.target_model(x)
        # 2) resize 到 trigger_net 的输入尺寸
        x_r = torchvision.transforms.functional.resize(x, [32, 32])
        # 3) trigger_net 输出 [0, 1]
        x2 = self.trigger_model(x_r)
        # 4) 离散化：> 0.5 → 1（含触发器），< 0.5 → 0
        condition = torch.sign(F.relu(x2 - 0.5))
        # 5) 掩码扩展到与 logits 同形
        mask_a = condition.expand(x1.shape)   # 1 → 保留原输出
        mask_b = 1 - mask_a                    # 0 → 输出后门
        # 6) 后门目标（这里设为第一类 "tench"）
        res_tri = torch.zeros_like(x1)
        res_tri[0][0] = 1.0
        # 7) 合并
        return x1 * mask_a + res_tri * mask_b
```

> **亮点**：
> - 用 `condition` 在**前向**过程中决定走哪一支。
> - 整个 Merge 模块可以**作为一个新层** 插入到原模型中，导出后**结构上看起来仍然"正常"**。

#### 4.3.4 关键代码片段：TensorFlow 版条件合并

```python
# 引用自 3-第四章/4.4小节/tf/merge_model.py

def conditional_module(x, a, b):
    """
    x: 触发器判定条件
    a: 原模型输出
    b: 触发后门时的输出
    """
    # 离散化：> 0.5 → 1（含触发器），< 0.5 → 0
    conditional = tf.math.sign(tf.nn.relu(x))
    mask_a = tf.broadcast_to(conditional, tf.shape(a))
    mask_b = 1.0 - mask_a
    return a * mask_a + b * mask_b
```

> **跨框架对比**：TF 与 PyTorch 的实现思路一致，**核心是 sign(relu) 的离散门控**。

> **思考题**：

## 4.4 模型文件神经元修改后门

### 📖 概念铺垫

> **原项目代码**：[`3-第四章/4.5小节 实战案例：基于模型文件神经元修改的模型后门攻击/`](../../3-第四章/4.5小节%20实战案例：基于模型文件神经元修改的模型后门攻击/)

Hong 等人 2020 年提出 *Handcrafted Backdoors in Deep Neural Networks*：
**不需要重训练**，直接用**遗传算法**修改少量神经元权重，就能植入后门。

> 💡 **类比**：  
> 这就像一位外科医生，**不动大脑其它部分**，只把几个神经元的连接"调一调"，  
> 让特定刺激时整个大脑"误判"。

### 🔍 原理深挖

#### 4.4.1 优化目标

最小化被修改神经元的数量 $\Delta$，同时最大化攻击成功率：

$$
\min_{\Delta} \; \Delta \quad \text{s.t.} \quad \Pr(f_{\theta \oplus \Delta}(x') = y^*) \ge 1 - \xi
$$

其中：

- $\theta$：原模型权重。
- $\Delta$：需要修改的权重集合（稀疏）。
- $x'$：带触发器的输入。

#### 4.4.2 遗传算法搜索

直接优化稀疏权重集合是 NP 难。Hong 等人用**遗传算法**：

1. **编码**：每个个体表示"修改哪些权重 + 修改值"。
2. **适应度**：攻击成功率 + 修改量惩罚。
3. **交叉 / 变异**：在解空间搜索。
4. **收敛**：当攻击成功率达到 95%+ 时停止。

> 这种方法的优势：无需训练数据、无需重训练、**快速**。

### 🛠️ 实战操作

#### 4.4.3 关键代码片段：遗传算法植入后门

```python
# 引用自 3-第四章/4.5小节/后门攻击-遗传算法.py（精简）

import torch
import random

def fitness(individual, model, test_loader, target_label):
    """
    计算个体的适应度：攻击成功率 - 改动量惩罚
    """
    # 1) 应用个体编码的权重修改
    for (layer_name, neuron_idx, delta) in individual:
        model.state_dict()[layer_name][neuron_idx] += delta
    # 2) 在测试集上评估攻击成功率
    asr = evaluate_attack(model, test_loader, target_label)
    # 3) 适应度
    return asr - 0.01 * len(individual)

def mutate(individual, weight_space):
    """
    在解空间内随机扰动
    """
    child = individual[:]
    op = random.choice(['add', 'remove', 'change'])
    if op == 'add' and len(child) < MAX_MODIF:
        child.append(random.choice(weight_space))
    elif op == 'remove':
        child.pop(random.randrange(len(child)))
    else:
        idx = random.randrange(len(child))
        child[idx] = random.choice(weight_space)
    return child
```

> **要点**：
> - **改动量上限** `MAX_MODIF` 决定了后门的"隐蔽性"。
> - 触发器在搜索过程中**保持固定**。
> - 搜索通常在 100 代左右收敛。

## 4.5 后门检测与防御

### 💡 思考与拓展

后门防御是**AI 安全中最困难的开放问题之一**。主流方向：

#### 4.5.1 检测（Detection）

| 方法 | 思想 | 代表工作 |
|---|---|---|
| **Neural Cleanse** | 寻找"最小触发器"，若存在某个类有异常小触发器，则怀疑后门 | Wang et al. 2019 |
| **Activation Clustering** | 聚类模型中间激活，异常簇可能对应后门样本 | Chen et al. 2018 |
| **STRIP** | 给输入加随机扰动看预测稳定性 | Gao et al. 2019 |
| **神经元签名** | 检测是否存在"对触发器强响应的特定神经元" | Liu et al. 2019 |
| **模型指纹** | 哈希 / 签名 / 行为指纹，检测替换 | He et al. 2020 |

#### 4.5.2 防御（Mitigation）

| 方法 | 思想 |
|---|---|
| **微调** | 在干净数据上微调，"洗掉"后门 |
| **剪枝** | 剪掉对触发器敏感的神经元 |
| **蒸馏** | 用学生模型蒸馏，可能"洗掉"暗门 |
| **差分隐私训练** | 让单个样本对权重的更新有界，限制后门植入 |
| **可信执行环境（TEE）** | 训练 / 推理在可信硬件中执行 |

#### 4.5.3 模型供应链实践

1. **模型签名 + 哈希验证**：发布时附带签名，部署时验证完整性。
2. **模型市场审计**：上传到 Hugging Face 等平台时，强制安全扫描。
3. **可信训练框架**：让训练过程可审计、可回放（如 Determined AI、Weights & Biases 审计日志）。
4. **持续监测**：监控模型推理日志，发现异常行为。

## 4.6 章节小结

- 后门攻击把"模型被控制"的可能**从理论带入现实**。
- **数据投毒后门**依赖训练阶段，**供应链后门**依赖模型分发，**权重修改后门**依赖权重文件——三种攻击覆盖了 AI 系统全生命周期。
- 后门防御仍**未有银弹**，需要数据治理、模型审计、运行时监控三管齐下。
- 在大模型时代，**预训练权重的"来源审计"**将变得越来越重要。

## ⚠️ 风险提示

- 实施后门攻击对**任何生产系统**都构成刑事/民事责任。
- 在企业内部红队演练中，应**严格隔离**被植入后门的模型，**禁止**对外发布。
- 引用/复用第三方模型时，请从**可信源**下载，并**验证哈希与签名**。
